# 🏭 エッジAI×メカトロ連携実習 コースプラン（完全決定版）

## 1. 概要・目的

本実習では、AIによる画像認識（エッジAI）と、生産設備制御（マイコン・PLC）を融合させた「スマート工場」の自動選別システムを構築する。
一昨年のテンプレートマッチング（OpenCV）のようなパラメータ調整の泥沼を排除し、現代的なディープラーニング（AIへのティーチング）アプローチを採用する。

通信経路には、実績があり最もシンプルな「Thonny REPL（標準入出力）」を流用。
また、最終回のレポート提出に向けて、毎回の実習成果をその都度ドキュメント（レポートの部品）として蓄積する「構造化ライティング指導」を全回で徹底する。

---

## 2. 開発・実習環境

### 🛠️ ハードウェア構成（1グループあたり）

* **メインPC（AI脳）：** 学生用ノートPC（Windows / Anaconda環境）
* **エッジAIカメラ（目）：** **OAK-D-Lite**（カラーカメラとして運用）
* **カメラ固定具：** **Ulanzi MT-08 三脚**（自由雲台を活用し、ワークの真横〜15度俯瞰で固定）
* **照明：** LEDリングライト（光環境の安定化用）
* **制御マイコン（手足）：** **Raspberry Pi Pico 2 WH**（PCからのThonny標準入力、リレー駆動）
* **絶縁インターフェース：** **M5Stack用ミニリレーユニット**（3.3V信号で24V外部回路を安全に開閉）
* **生産設備（結合テスト用）：** ピック＆プレース付きコンベアライン（PLC 24V制御 / メカトロ実習室）
* **検査対象（ワーク）：** 3部品構成の円筒メタリックパーツ（赤・青・黄のランダムカラー、1cm径、専用の嵩上げパレット付き）

### 💻 ソフトウェア・Web環境

* **Python環境：** Anaconda（またはMiniconda）
* **マイコン開発環境：** Thonny (MicroPython)
* **PC側制御・画像処理：** Python 3.10, DepthAI SDK, OpenCV-Python（※`pyserial`は不要）
* **データセット・AI学習：** **Roboflow**（無料アカウント、ブラウザ上でアノテーション・YOLOモデル学習）

## 3. 全8回 コーススケジュール（各200分）

※各回、最後の15分を「ドキュメント（レポートの部品）作成時間」として固定する。

### ■ フェーズ1：別ロケーション開発（情報教室で完結）

#### 🟢 第2回：システムの中枢神経（リレー制御）＋ AI環境の先行構築【前倒し】

* **実習内容（前半）：**
* Thonnyを使用し、PC側の `print()` からThonny経由でPico 2 WHへ信号を送り、Pico側は `input()` でそれを受け取るプログラムを作成。
* PicoのGPIOからM5Stackミニリレーユニットを叩き、「カチッ」という物理的な動作音とLED点灯を確認する。


* **実習内容（後半：第3回から前倒し）：**
* 学生PCにAnaconda環境を構築し、競合を防ぐための専用仮想環境（`oak_env`, Python 3.10）を作成・アクティベート。
* `pip install depthai-sdk opencv-python` で必要なライブラリを先行導入（ネットワーク混雑対策）。


* **📝 毎回の仕込み（残り15分）：**
* 「通信成功時のPython/MicroPythonコード」と「リレーの写真」を記録。
* 『Thonnyの標準入出力を流用してPCから物理世界へ信号が伝わる仕組み』を箇条書きでメモさせる。



#### 🟢 第3回：AIカメラ接続テストとアングル調整の極限化

* **実習内容：**
* 前回構築した仮想環境下でOAK-D-Liteを接続し、数行のテストコードでカラー映像がPC画面に常時ストリーミング表示されることを確認。
* 三脚とリングライトを設置し、ワークのハメ合い（浮き・傾き）が最もよく見えるアングル（真横〜15°俯瞰）をじっくりと調整・固定する。
* ワーク（赤・青・黄）を実際に動かし、どのように回転させたら不具合（浮き）がシルエットに現れるか予備実験を行う。


* **📝 毎回の仕込み（残り15分）：**
* 「映像が表示された画面のスクリーンショット」と「決定したカメラのアングル写真」を保存。
* なぜそのアングル（真横〜15°）にしたのか、『手前のレールの回避や、パーツの死角をなくすための理由』を論理的に記述させる。



#### 🟢 第4回：データ収集とAIへのティーチング（Roboflow）

* **実習内容：**
* メカトロ実習室から「嵩上げパレット」とワーク一式を一時的に借りてくる。
* 情報教室の机の上に「均一な背景（白布または紙）」を設置し、固定したカメラの前で、ワークを少しずつ回転させながら大量の写真を撮影（1班あたり100〜200枚）。
* 状態パターン（`OK`、アジャストミスの `NG_Uki`、部品欠品の `NG_Miss`）をランダムカラーを交えて網羅し、Roboflowにアップロード。アノテーション（ラベル付け）を行い、クラウドでAIモデルの学習を開始する。


* **📝 毎回の仕込み（残り15分）：**
* Roboflowの「アノテーション画面のスクリーンショット」を保存。
* AIに賢く学習させるために『撮影時に工夫したこと（回転させる、色のバラつき、背景を単色に統一する目的等）』を記述させる。



#### 🟢 第5回：脳と神経の結合（情報教室内でのシステム単体完成）

* **実習内容：**
* 学習が完了したAIモデルをダウンロードし、OAK-D-Liteに読み込ませてリアルタイム常時監視（動画ループ）を行う。
* 「画面内に信頼度85%以上で `NG_Uki` または `NG_Miss` が映ったら、即座にPicoへ信号を送る」という常時監視型ロジックのPythonコードを作成。
* 手元でワークを動かし、NG状態を作ると即座にリレーが「カチッ」と鳴る単体完結システムを完成させる。


* **📝 毎回の仕込み（残り15分）：**
* 「合体させたPythonのメインソースコード」を保存。
* タイミングトリガー（外部センサ）なしの「常時監視型ロジック（if文）」を採用したメリット（タイミングのズレによる検知ミスが起きない等）を記述させる。

### ■ フェーズ2：異電圧インターフェースと安全設計

#### 🟢 第6回：生産設備（24V）への架け橋と回路構築

* **実習内容：**
* M5リレーユニットの出力側端子台（COM / NO）に配線を行い、テスター（マルチメーター）を用いて導通チェック（抵抗値・ピー音）を実施。
* マイコン側の低電圧（3.3V）回路と、PLC側の高電圧（24V）回路が、リレー内部で電気的に「完全に絶縁」されている仕組みを回路図を通して学ぶ。


* **📝 毎回の仕込み（残り15分）：**
* 「自分たちで配線したリレーまわりの写真」を撮影。
* 『なぜリレーを挟む必要があるのか（異電圧結合における絶縁の重要性）』を安全面の観点から説明させる。



---

### ■ フェーズ3：コンベアラインとの統合（メカトロ実習室への出張）

#### 🟢 第7回：メカトロラインとの物理ドッキング（結合テスト）

* **実習内容：**
* メカトロ実習室へ移動（他科の先生の授業の隙間に最短時間で割り込み）。
* コンベア横に三脚を立て、ワークの奥側に「撮影時と同じ均一な背景布/紙」を配置。リレーの出力線をPLCの入力端子へネジ留め接続する。
* 実際にコンベアを動かし、不良ワークが通過した際にPLCの入力ランプが点灯し、排出シリンダが連動してワークを弾くかを確認する。


* **📝 毎回の仕込み（残り15分）：**
* 「コンベア横にドッキングしたシステム全体の様子」を写真に収める。
* 結合テストで一発で動いたか、あるいはどんな物理的な問題が起きたかのファクトをメモさせる。



#### 🟢 第8回：外乱チューニングと総合デバッグ

* **実習内容：**
* 実稼働時のスピードにおいて、検知漏れや誤動作がないかを連続テスト。
* 「工場の光環境による誤検知」などの問題に対し、プログラム側の信頼度閾値（Confidence）の微調整や追加撮影を行い、ライン全体の検知率100%を目指す。


* **📝 毎回の仕込み（残り15分）：**
* 『発生したトラブルと、それをハード（照明・アングル）またはソフト（閾値・再学習）のどちらで解決したか』の対照表をメモさせる。（※レポートの「考察」の核になります）



#### 🟢 第9回：最終稼働デモンストレーションと構造化レポート

* **実習内容：**
* 担当教員の前で自動仕分けラインの最終デモ走行を実施（良品はスルー、不良品はシリンダで確実に排出）。
* **【レポート執筆】** 第2回〜第8回まで毎回書き溜めてきた「仕込みメモ」と「写真」をパズルのように組み合わせ、指定のフォーマット（背景・目的・システム構成・結果・考察）に構造化して技術レポートを完成させ、その場で提出する。
